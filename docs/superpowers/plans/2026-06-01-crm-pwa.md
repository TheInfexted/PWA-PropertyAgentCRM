# PWA — Installable + App-Shell Caching Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the CRM an installable PWA with app-shell (static asset) caching, fulfilling spec §5 ("PWA — `@vite-pwa/nuxt` — installable + app-shell caching").

**Architecture:** Add `@vite-pwa/nuxt` with a web manifest, brand icons, and a Workbox-generated service worker that precaches the built client assets (JS/CSS/fonts/icons) and runtime-caches Google Fonts. Because the app is **server-rendered**, we deliberately do NOT set `navigateFallback` — online navigations must reach the Nitro server, and "offline-first data sync" is explicitly out of scope (spec §2). So the PWA delivers fast repeat loads + installability, not offline data.

**Tech Stack:** Nuxt 4, `@vite-pwa/nuxt` 1.1.1 (Workbox `generateSW`), `rsvg-convert` (local, for icon rasterization only — not a runtime/deploy dependency).

**Plan:** Plan 8 of the Phase-1 sequence (last of three "finish Phase 1" plans). Branch: `feat/pwa`.

## Task 1: Brand icons

**Files:** Create `public/icon.svg`, `public/pwa-192x192.png`, `public/pwa-512x512.png`, `public/maskable-icon-512x512.png`, `public/apple-touch-icon-180x180.png`.

- [ ] **Step 1: Author the source SVGs** — a terracotta (`#9e5733`) square with the white house mark from `public/favicon.svg`.
  - `public/icon.svg`: rounded square (`rx="112"`), house at `translate(32,34) scale(14)`.
  - `/tmp/icon-maskable.svg`: full-bleed square, house at `translate(56,58) scale(12.5)` (content within the maskable safe zone).

- [ ] **Step 2: Rasterize with `rsvg-convert`:**
```bash
rsvg-convert -w 192 -h 192 public/icon.svg -o public/pwa-192x192.png
rsvg-convert -w 512 -h 512 public/icon.svg -o public/pwa-512x512.png
rsvg-convert -w 512 -h 512 /tmp/icon-maskable.svg -o public/maskable-icon-512x512.png
rsvg-convert -w 180 -h 180 /tmp/icon-maskable.svg -o public/apple-touch-icon-180x180.png
```

- [ ] **Step 3: Verify dimensions** (`sips -g pixelWidth -g pixelHeight <file>`) and visually confirm the house mark renders on terracotta.

- [ ] **Step 4: Commit**
```bash
git add public/icon.svg public/pwa-192x192.png public/pwa-512x512.png public/maskable-icon-512x512.png public/apple-touch-icon-180x180.png
git commit -m "feat(pwa): brand app icons (192/512 + maskable + apple-touch)"
```

## Task 2: Install module + configure manifest/service worker

**Files:** Modify `nuxt.config.ts`, `package.json`, `pnpm-lock.yaml`.

- [ ] **Step 1:** `pnpm add -D @vite-pwa/nuxt`.

- [ ] **Step 2: Register the module + configure** in `nuxt.config.ts`:
  - Add `'@vite-pwa/nuxt'` to `modules`.
  - Fix the stale `theme-color` head meta from `#0d121c` → `#9e5733`; add `{ rel: 'apple-touch-icon', href: '/apple-touch-icon-180x180.png' }`.
  - Add a `pwa` block:
    - `registerType: 'autoUpdate'`.
    - `manifest`: name/short_name/description, `theme_color: '#9e5733'`, `background_color: '#faf8f4'`, `display: 'standalone'`, `start_url: '/'`, `scope: '/'`, and the four icon entries (192, 512, 512 `any`, 512 `maskable`).
    - `workbox`: `globPatterns: ['**/*.{js,css,html,svg,png,ico,woff,woff2}']`; `runtimeCaching` for `fonts.googleapis.com` (StaleWhileRevalidate) and `fonts.gstatic.com` (CacheFirst, 1-year). **No `navigateFallback`** (SSR-safe).
    - `client: { installPrompt: true }`, `devOptions: { enabled: false }`.

- [ ] **Step 3: Typecheck + build + verify artifacts**
```bash

```
Expected: `.output/public/` contains `sw.js`, `workbox-*.js`, `manifest.webmanifest`, and all icon PNGs; the manifest JSON has the correct name/theme/icons.
